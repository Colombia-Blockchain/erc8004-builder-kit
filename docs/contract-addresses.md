## CAIP-10 Identifiers

The CAIP-10 format for referencing registries across chains:

```
eip155:<chainId>:<registryAddress>
```

**Examples:**

- Ethereum Identity Registry: `eip155:1:0x8004A169FB4a3325136EB29fA0ceB6D2e539a432`
- Base Identity Registry: `eip155:8453:0x8004A169FB4a3325136EB29fA0ceB6D2e539a432`
- Avalanche Identity Registry: `eip155:43114:0x8004A169FB4a3325136EB29fA0ceB6D2e539a432`

## Vanity Address Pattern

All ERC-8004 contracts share the `0x8004` prefix:

| Contract | Prefix | Purpose |
|---|---|---|
| Identity Registry | `0x8004A...` | Agent registration and NFT ownership |
| Reputation Registry | `0x8004B...` | Feedback and reputation scores |
| Validation Registry | `0x8004C...` | Third-party validation requests/responses |

These vanity addresses are achieved through CREATE2 deterministic deployment, ensuring the same addresses across all EVM chains.
